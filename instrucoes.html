ENTROPIA

Guia de primeiro acesso ao servidor
"O Caos Sempre Reina."

Este documento te guia pelos passos necessários pra entrar no servidor ENTROPIA pela primeira vez, cadastrar os usuários da mesa e começar a jogar. Tempo estimado: 10 minutos.

Você vai precisar de:

  • Um cliente SSH (Terminal no macOS/Linux ou PowerShell no Windows 10+).
  • O IP e usuário do servidor (fornecidos separadamente).
  • A pasta do projeto ENTROPIA no seu computador pra rodar o cliente Tkinter.

1.Conectar no servidor via SSH

Abra o terminal da sua máquina (PowerShell no Windows, Terminal no macOS/Linux) e conecte:

ssh enzorpg@212.85.19.190

Na primeira conexão o SSH vai perguntar:

The authenticity of host '212.85.19.190' can't be established.
Are you sure you want to continue connecting (yes/no/[fingerprint])?

Digite yes e aperte Enter. Depois vai pedir a senha do usuário enzorpg. Digite (ela não aparece na tela — isso é normal) e aperte Enter.

Se deu certo, você verá uma tela de boas-vindas do CloudPanel e o prompt:

enzorpg@PortainerDesign:~$

Esse $ no final indica que você está dentro do servidor. Todos os comandos a seguir são digitados aqui, não no seu computador.

2.Entrar na pasta do projeto

cd ~/htdocs/rpg.enzobr.com

O prompt agora fica:

enzorpg@PortainerDesign:~/htdocs/rpg.enzobr.com$

3.Confirmar que o servidor está rodando

Antes de cadastrar usuários, vale testar se a API está no ar:

curl http://127.0.0.1:3013/saude

Resposta esperada:

{"ok": true, "dados_dir": "/home/enzorpg/htdocs/rpg.enzobr.com/dados"}
Se a resposta acima não aparecer, rode systemctl --user status entropia pra ver o status do service. Se estiver parado, suba com systemctl --user start entropia.

4.Cadastrar os usuários da mesa

O servidor tem seu próprio banco de usuários (senhas ficam hasheadas com bcrypt, seguras mesmo se o arquivo vazar). Rode o comando abaixo uma vez para cada jogador — incluindo o Mestre:

ENTROPIA_DADOS_DIR=$PWD/dados venv/bin/python servidor.py criar-usuario

O script é interativo. Responda cada pergunta e aperte Enter:

=== Criar usuario ENTROPIA ===
Usuario: [nome do personagem/jogador, ex: O MESTRE]
Senha: [digite a senha — nao aparece na tela, e normal]
Palavra-chave: [digite a palavra-chave — nao aparece]
Papel (mestre/jogador) [jogador]: mestre ou jogador
usuario 'O MESTRE' criado (mestre).

A senha e a palavra-chave não aparecerem quando digitadas é intencional (igual sudo). Apenas digite e aperte Enter.

Exemplo — cadastrando a mesa inteira

Rode o comando 5 vezes, respondendo com:

Usuário Papel
O MESTREmestre
AMÍLIAjogador
DANTEjogador
WYATTjogador
GUIZOjogador

Senhas e palavras-chave ficam a critério da mesa.

Listar quem foi cadastrado

venv/bin/python servidor.py listar-usuarios

Saída esperada:

  O MESTRE              mestre
  AMÍLIA                jogador
  DANTE                 jogador
  WYATT                 jogador
  GUIZO                 jogador

Remover um usuário (se precisar refazer)

venv/bin/python servidor.py remover-usuario

5.Sair do servidor

Com os usuários cadastrados, o trabalho no SSH acabou. Pode sair:

exit

Todo o resto acontece no computador de cada jogador, rodando o cliente Tkinter.

6.Configurar o cliente no computador do jogador

Cada jogador precisa ter a pasta do projeto ENTROPIA na máquina dele. Dentro dela, editar o arquivo app.json:

{
  "modo": "remoto",
  "servidor_url": "https://rpg.enzobr.com",
  "token": null,
  "usuario": null
}

Esse arquivo diz pro cliente falar com o servidor em vez de salvar tudo na máquina.

7.Rodar o cliente

Pré-requisitos (uma vez só)

  • Python 3.10+ instalado (python.org/downloads) — na tela de instalação, marque "Add Python to PATH".

Primeira vez — criar o ambiente virtual

No PowerShell, dentro da pasta do projeto:

py -3 -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt

Se o PowerShell reclamar de "execução de scripts desabilitada", rode uma vez: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned e tente de novo.

Abrir o jogo

python entropia.py

A tela de login aparece. Usa o mesmo usuário, senha e palavra-chave que foram cadastrados no passo 4.

Das próximas vezes

.\venv\Scripts\Activate.ps1
python entropia.py

8.Problemas comuns

Sintoma Causa provável Solução
Não conecta no SSH Senha errada ou IP errado Confirme IP e senha com quem te passou o acesso
curl /saude não responde Service caído systemctl --user start entropia
Login no cliente dá "usuário não encontrado" Usuário não foi cadastrado (passo 4) Volta ao SSH e rode o passo 4 pro usuário faltante
Cliente reclama de conexão URL errada em app.json Confira: deve ser https://rpg.enzobr.com
"Not Found" abrindo a URL no navegador Comportamento normal O servidor não serve HTML, só API. O cliente Tkinter é quem usa
